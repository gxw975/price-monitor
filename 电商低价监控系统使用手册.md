## 1. 系统整体架构说明

本系统专为分销商价格管控设计，采用纯真人浏览器模拟技术，100% 绕过淘宝反爬，无需任何无头浏览器。



| 模块   | 功能说明                           |
| ---- | ------------------------------ |
| 采集模块 | 定时抓取淘宝商品 SKU、价格、销量数据           |
| 存储模块 | PostgreSQL 持久化商品、SKU、历史价格、预警数据 |
| 预警模块 | 低价预警 / 销量突增预警，工作时段飞书推送         |
| 后台管理 | 前端页面管理商品、查看预警、数据导出             |
| 运维模块 | 自动部署、定时任务、日志轮转、数据库自动备份         |

## 2. 前期基础配置（首次部署必做）

### 2.1 飞书推送配置



1. 编辑环境变量文件：



```
nano /home/lab-admin/price-monitor/.env
```



1. 填入飞书机器人 Webhook 地址：



```
FEISHU\_WEBHOOK\_URL=https://open.feishu.cn/open-apis/bot/v2/hook/你的Webhook地址
```



1. 保存退出：`Ctrl+O` → 回车 → `Ctrl+X`

> **说明**
>
> ：未配置时仅记录日志，不会发送消息；非工作时段 (9:00-18:00) 自动跳过推送。

### 2.2 系统阈值配置

在数据库 `SystemConfig` 表配置全局预警阈值：



* `alert_price`：低价预警门槛（低于该价格触发提醒）

* `sales_growth_threshold`：销量涨幅阈值（百分比）

* `sku_crawl_limit`：每日最大 SKU 抓取数量

* `sku_crawl_interval`：商品间抓取间隔（秒）

### 2.3 浏览器登录（防爬必备）

在服务器桌面打开 Chrome 浏览器，**扫码登录淘宝账号**，保持浏览器后台运行，不要退出或关闭。

> 重要：OpenCLI 会自动保留登录状态，只要不手动退出账号，后续所有抓取都会使用已登录的会话，绕过 90% 以上的淘宝风控。

## 3. 日常操作全流程

### 3.1 服务启停 & 状态查看

#### 查看服务状态



```
sudo systemctl status price-monitor
```

#### 手动重启服务（更新代码 / 配置后）



```
cd /home/lab-admin/price-monitor

scripts/deploy.sh
```

#### 手动停止 / 启动服务



```
sudo systemctl stop price-monitor

sudo systemctl start price-monitor
```

### 3.2 商品管理流程

#### 步骤 1：导入待监控商品

通过后台前端或脚本批量导入商品信息，必填字段：



* `product_id`：淘宝商品 ID（唯一标识）

* `title`：商品名称

* `shop_name`：店铺名称

#### 步骤 2：商品审核

进入前端后台，对导入商品执行审核操作：



* ✅ **审核通过**：`is_approved = true`，纳入监控范围

* ⚪ **加入白名单**：`is_whitelist = true`，不触发低价预警

* ❌ **拒绝**：删除该商品及所有相关记录

> **核心规则**
>
> ：仅「已审核 + 非白名单」商品参与价格和销量预警检测。

### 3.3 手动触发 SKU 抓取

#### 抓取单个商品



```
.venv/bin/python src/scripts/run\_sku\_crawl.py --product-id 淘宝商品ID
```

#### 批量抓取（按系统默认规则）

自动跳过 24 小时内已抓取的商品：



```
.venv/bin/python src/scripts/run\_sku\_crawl.py
```

#### 查看采集日志



```
\# 实时查看最新日志

tail -f logs/crawl.log

\# 查看历史日志

cat logs/crawl\_YYYYMMDD.log
```

### 3.4 预警检测手动执行

#### 测试模式（仅检测 + 入库，不发飞书）



```
.venv/bin/python src/scripts/check\_alerts.py --test
```

#### 正常模式（检测 + 入库 + 符合条件则推送飞书）



```
.venv/bin/python src/scripts/check\_alerts.py
```

#### 强制模式（忽略 24 小时去重限制，强制检测推送）



```
.venv/bin/python src/scripts/check\_alerts.py --force
```

#### 查看预警日志



```
\# 实时查看最新日志

tail -f logs/alert.log

\# 查看历史日志

cat logs/alert\_YYYYMMDD.log
```

## 7. 常见问题排查

### 7.1 抓取提示「无待抓取商品」

**原因**：商品 24 小时内已抓取，命中去重规则

**解决**：等待下一轮定时，或使用 `--product-id` 参数单独抓取

### 7.2 抓取返回「SKU 为空」

**原因**：商品为单规格，无多 SKU 选项，属于正常现象

**说明**：系统会自动降级为 "默认规格 + 当前价格"

### 7.3 预警检测结果为 0

**可能原因**：



1. 暂无商品低于价格阈值或销量无明显涨幅

2. 商品未审核或已加入白名单，不参与预警

3. 系统中未导入任何监控商品

### 7.4 飞书收不到消息

**排查步骤**：



1. 检查 `.env` 内 `FEISHU_WEBHOOK_URL` 是否正确

2. 检查当前时间是否在工作时段 9:00-18:00

3. 执行 `--test` 模式验证脚本本身无报错

4. 检查飞书机器人是否在目标群聊中

### 7.5 定时任务不执行

**排查步骤**：



1. 检查 cron 服务状态：`sudo systemctl status cron`

2. 检查脚本绝对路径是否与当前服务器路径一致

3. 手动执行对应脚本，确认脚本本身可运行

4. 查看 cron 系统日志：`tail -f /var/log/cron.log`

### 7.6 数据库恢复



```
pg\_restore -U postgres -d openclaw -n price\_monitor 备份文件路径.dump
```
